## 4. 상세 To-Do 리스트

### 4.1 프로젝트 생성

-  Node.js LTS 버전 확인
    
-  Next.js 프로젝트 생성
    
    - TypeScript 사용
        
    - App Router 활성화
        
-  프로젝트 실행 확인 (`localhost`)
    

**완료 기준**

- 기본 Next.js 화면 정상 출력
    

---

### 4.2 Git & 저장소 설정

-  Git 초기화
    
-  `.gitignore` 확인
    
-  GitHub 원격 저장소 생성
    
-  main 브랜치 push
    

**완료 기준**

- GitHub 저장소와 로컬 연동 완료
    

---

### 4.3 기본 폴더 구조 정리

-  `src/` 구조 확인
    
-  `app/` 디렉토리 확인
    
-  불필요한 샘플 코드 제거
    
-  기본 파일 정리
    

```
src/
 ├ app/
 │   ├ layout.tsx
 │   ├ page.tsx
 │   └ globals.css
```

**완료 기준**

- 최소 구조로 정리됨
    

---

### 4.4 글로벌 스타일 설정

-  `globals.css` 작성
    
-  CSS Reset 적용
    
-  기본 폰트 설정
    
-  body 기본 스타일 정의
    

**완료 기준**

- 모든 페이지에 스타일 일관 적용
    

---

### 4.5 기본 레이아웃 구성

-  `layout.tsx` 수정
    
-  Header 컴포넌트 생성
    
-  Footer 컴포넌트 생성
    
-  children 영역 정상 렌더링
    

**완료 기준**

- 모든 페이지에 Header/Footer 표시
    

---

### 4.6 테스트 페이지 구현

-  `/page.tsx` 수정
    
-  임시 텍스트 콘텐츠 작성
    
    - 페이지 제목
        
    - 간단한 설명
        
-  임시 버튼 추가
    

예:

- “Portfolio Test Page”
    
- “Deployment is working”
    

**완료 기준**

- 테스트 콘텐츠 화면 노출
    

---

### 4.7 TypeScript 기본 설정 확인

-  tsconfig 설정 검토
    
-  strict 모드 확인
    
-  타입 오류 없는지 확인
    

**완료 기준**

- 빌드 에러 없이 실행
    

---

### 4.8 ESLint / 코드 품질 (선택)

-  ESLint 기본 설정 확인
    
-  경고/에러 확인
    
-  불필요한 규칙 제거
    

**완료 기준**

- 개발 시 오류 최소화
    

---

### 4.9 Vercel 배포 설정

-  Vercel 계정 연결
    
-  GitHub 저장소 연동
    
-  배포 프로젝트 생성
    
-  최초 자동 배포 실행
    

**완료 기준**

- Vercel URL 정상 접속

### 4.10 배포 검증

-  데스크톱 브라우저 확인
