디자인/개발/학생 포트폴리오 어디에도 무리 없이 적용할 수 있게 정리했어요.

---

## 1. 기획 목표

- **한눈에 정체성 전달**: “나는 누구이고 무엇을 하는 사람인가”
    
- **불필요한 요소 제거**: 콘텐츠 중심, 여백 활용
    
- **가독성 최우선**: 빠르게 읽히고 이해되는 구조
    
- **확장 가능성**: 프로젝트 추가가 쉬운 구조
    

---

## 2. 전체 콘셉트

- 키워드: **Minimal / Simple / Clean / Focus**
    
- 장식 최소화, 텍스트와 콘텐츠가 주인공
    
- 애니메이션은 **있어도 아주 미세하게** (hover, fade 정도)
    

---

```
Home
```

---

## 4. 페이지별 구성

### 1) Home (메인)

**목적:** 첫인상 + 정체성

- 간단한 한 줄 소개
    
    > “미니멀한 디자인을 좋아하는 웹 개발자”
    
- 이름 / 역할
    
- 스크롤 유도 (↓ 아이콘 또는 “Scroll”)
    

**포인트**

- 화면 중앙 정렬
    
- 배경은 단색 (화이트 or 라이트 그레이)
    

---

### 2) About

**목적:** 나에 대한 이해

- 짧은 자기소개 (3~5줄)
    
- 관심 분야
    
- 가치관 또는 작업 스타일
    

예:

- 심플한 문제 해결을 좋아합니다
    
- 사용자 경험 중심 설계
    

---

### 3) Projects

**목적:** 핵심 콘텐츠

- 프로젝트 카드 형태 (세로 정렬)
    
- 각 프로젝트:
    
    - 제목
        
    - 한 줄 설명
        
    - 역할
        
    - 사용 기술
        
    - 링크 (선택)
        

**디자인**

- 테두리 없는 카드
    
- hover 시 얇은 라인 또는 색 변화
    

---

### 4) Skills

**목적:** 역량 한눈에

- 아이콘 + 텍스트
    
- 카테고리별 정리
    
    - Frontend
        
    - Design
        
    - Tools
        

예:

- HTML / CSS / JavaScript
    
- Figma / Git
    

---

### 5) Contact

**목적:** 연결

- 이메일
    
- GitHub / Notion / Instagram 등
    
- 짧은 문장
    
    > “언제든 편하게 연락 주세요.”

## 5. 디자인 가이드

### 컬러

- 메인: `#FFFFFF` / `#F5F5F5`
    
- 포인트: 블랙 또는 한 가지 색만  
    예: `#222222` or `#4A6CF7`
    

### 타이포그래피

- 산세리프 폰트
    
    - Noto Sans KR
        
    - Pretendard
        
- 제목: Bold
    
- 본문: Regular
